# Сводка ҚБМ

PWA-дашборд по основным показателям АО «Қаражанбасмұнай». Открывается утром на телефоне — сразу видишь ключевые цифры за сутки.

## Что показывает

- **Добыча Qн за сутки** + дельта к вчера + Qн факт
- **KPI-сетка:** Сдача · Жидкость на ДНС + обводнённость · Технологические потери · Остаток в парке
- **Накопительно за месяц:** добыча, сдача, средне в сутки, прогноз на месяц
- **График добычи по дням** текущего месяца
- **Пар** (ППГ-3 / СПГУ / МПГУ), **закачка воды**, **газ**, **ОЗПВ** — за месяц
- **Фонд скважин:** действующий доб. (фонтан + ШГН), в простое, нагнетательный (пар + вода)

## Установка на телефон (PWA)

1. Залить файлы в репо `kbm-svodka` на GitHub (юзер `maratkor`)
2. В Settings → Pages включить Pages с ветки main / root
3. На телефоне открыть `https://maratkor.github.io/kbm-svodka/`
4. Android Chrome → меню → «Добавить на главный экран»
   iPhone Safari → Поделиться → «На экран Домой»

## Источник данных — 4 варианта

### Вариант 1 (рабочий с первого дня): ручная загрузка
- Нажимаешь 📂 в шапке → выбираешь XLSX-файл
- Парсится в браузере (SheetJS), сохраняется в localStorage
- Работает с файлом из любого источника: почта, Telegram, флешка, корп. диск
- **Минус:** утром нужно вручную скачать файл и подгрузить

### Вариант 2 (полу-авто): GitHub + ежедневный git push  ⭐ рекомендую
1. На рабочем компе ставишь bat/PowerShell-скрипт, который каждое утро берёт свежий файл с корп. сервера и пушит в **приватный** репо `kbm-svodka-data`
2. Раз создаёшь Personal Access Token в GitHub
3. В меню приложения → «Настроить авто-синхронизацию» → вставляешь raw-ссылку вида:
   `https://raw.githubusercontent.com/maratkor/kbm-svodka-data/main/svodka.xlsx`
4. Жмёшь ↻ — приложение само скачивает свежий файл и обновляет дашборд
- **Плюс:** работает на 100%, CORS нет, версионирование бесплатно
- **Минус:** разово настроить скрипт на ПК

Скрипт PowerShell (пример) — `C:\Scripts\push-svodka.ps1`:
```powershell
$src = "\\server\share\Сводка КБМ 2026.xlsx"
$dst = "C:\Users\Marat\repo\kbm-svodka-data\svodka.xlsx"
Copy-Item $src $dst -Force
cd C:\Users\Marat\repo\kbm-svodka-data
git add svodka.xlsx
git commit -m "auto: $(Get-Date -Format 'yyyy-MM-dd HH:mm')"
git push
```
Поставить в Планировщик задач Windows — запуск каждый день в 09:00.

### Вариант 3: OneDrive / SharePoint direct link
Если файл лежит на корпоративном OneDrive с публичным шарингом:
1. Получаешь ссылку «Поделиться» → копируешь
2. Преобразуешь в direct-download: добавляешь `&download=1` в конце
3. Вставляешь в меню «Настроить авто-синхронизацию»
- **Минус:** OneDrive может блокировать CORS из браузера → не всегда работает

### Вариант 4: Google Drive direct link
1. Файл → Поделиться → «Доступ по ссылке для всех»
2. Из ссылки `https://drive.google.com/file/d/{FILE_ID}/view` берёшь FILE_ID
3. Direct-download: `https://drive.google.com/uc?export=download&id={FILE_ID}`
4. Вставляешь в меню
- **Минус:** для больших файлов гугл вставляет страницу-предупреждение → не сработает напрямую

## Что я рекомендую

**Вариант 2** — GitHub приватный репо + PowerShell-скрипт по расписанию. Один раз настроить за полчаса, дальше работает само. Утром открыл приложение → жмёшь ↻ → актуальные цифры на экране. Если нет рабочего ПК под рукой → вариант 1, тоже работает.

## Структура файла

Парсер читает 3 листа:
- `1-Мәліметтер` — суточные данные (12 строк параметров × 31 день)
- `Показатели` — месячные сводки
- `Сут.рапортАО КБМ` — фонд скважин (фонтан/ШГН/простой/ППД)

Если на корп. сервере поменяется структура листов — нужно будет обновить `PARAM_ROWS` и `SUMMARY_ROWS` в `app.js`.

## Технологии
- Vanilla JS, без фреймворков
- [SheetJS](https://sheetjs.com) для парсинга XLSX в браузере
- Canvas API для графика (без Chart.js — экономия трафика)
- localStorage для кэша
- PWA: manifest.json + иконки
